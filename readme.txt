Клиентская часть
файл registation.html: после заполнения полей и нажатия кнопки отправки, данные отправляются на сервер, 
там идет проверка на существование никнейма, и, если такого нет, пользователь регистрируется
файл login.html: после ввода данных они идут на проверку на сервер, если данные неверны, это сообщается, если верны, производится переход в чат
файл chat.html: после попадания производится проверка, зашел пользователь после ввода логина, или просто по адресу, в случае перехода по адресу осуществляется переброс в login.html,
в самом чате есть отображения количества пользователей и их логины, при отправке сообщения это сообщение отправляется другим пользователям с указанием времени отправки,
при чем у отправившего ровнение на правый край, а у остальных на правый

Серверная часть
Сервер написан в классе WS_Server. В нем есть контейнеры, в которых хранятся данные подключенных пользователей. В случае закрытия соединения это сообщается другим пользователям.
Все события обрабатываются посредством получения сообщений от пользователей различных форматов. Есть форматы: регистрация, вход, подключение к чату и сообщение чата.


запуск (из директории): java -cp "jetty.jar" Server.java
